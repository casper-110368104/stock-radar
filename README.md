# Stock Radar 台股選股雷達

每日自動掃描台股，整合籌碼、基本面、量能、營收、相對強弱五大面向，給出綜合評分與買點訊號。

---

## 資料來源

| 資料 | 來源 | 說明 |
|------|------|------|
| 每日量價、類股指數 | TWSE 證交所公開 API | 免費，無須 token |
| 股價歷史、基本面 | Yahoo Finance (yfinance) | 免費，無須 token |
| 三大法人籌碼 | FinMind API | 免費個股查詢，需 token |
| 借券資料 | TWSE TWT93U | 免費，無須 token |
| 月營收 YoY | MOPS 公開資訊觀測站 / OpenData | 免費，無須 token |
| 新聞情緒 / 題材 | Google Gemini API | 需 API key，僅處理前 20 名個股 |

## 評分系統

每支股票的各面向分數皆為 **0～100 分**，最終再依下列權重加總為**綜合分數**：

```
綜合分數 = 籌碼面×33% + 基本面×28% + 量能面×23% + AVWAP對齊×6% + 月營收×5% + 相對強弱×5%
```

### 籌碼面（Chips）— 佔 33%

**基礎分 50 分**，視三大法人動向加減分。

#### 外資

|------|---------|

| 量比（vr） | 分數 |
|-----------|------|
| > 2.0 | 100 |
| > 1.5 | 85 |

| > 1.1 | 55 |
| > 1.0 | 40 |
| > 0.8 | 25 |
| ≤ 0.8 | 10 |

> 若無法取得60日均量，改以當日量 vs 月均量（vol_month_ratio）代替，換算邏輯相同。

### AVWAP 對齊（Anchored VWAP）— 佔 6%

統計股價目前站在幾條 AVWAP 之上（0～3 條），反映主力浮盈結構與短線節奏健康度：

| 對齊條數 | 分數 |
|---------|------|
| 3 條全對齊（起漲 + 主力 + 短線） | 15 → 換算 100 |
| 2 條對齊 | 10 → 換算 67 |
| 1 條對齊 | 5 → 換算 33 |
| 0 條（全部破線） | 0 |

> 三條全對齊代表：趨勢未壞（起漲 AVWAP）、主力未跑（主力 AVWAP）、短線洗盤結束（短線 AVWAP）。

|--------|------|
| > 40% | 100 |
| > 20% | 85 |
| > 10% | 70 |
| > 5% | 60 |
| > 0% | 52 |
| > -5% | 45 |
| > -20% | 30 |

### 相對強弱 RS（Relative Strength）— 佔 5%

與大盤比較的百分位排名（0～100）：

| RS 百分位 | 分數 |
|-----------|------|
| ≥ 90 | 100 |
| ≥ 80 | 85 |
| ≥ 70 | 70 |
| ≥ 50 | 55 |
| ≥ 30 | 40 |
| < 30 | max(10, 百分位 × 0.8) |

> 若無百分位資料但有 RS 絕對值，改以 RS 值換算（±0.20 對應 85/10 分）。

## 技術指標說明

### 均線（Moving Average）

| 均線 | 意義 |
|------|------|
| MA5 | 5日短期趨勢 |
| MA10 | 10日短期趨勢 |
| MA20 | 20日中期趨勢（月線） |
| MA60 | 60日中長期趨勢（季線） |

**多頭排列**：MA5 > MA10 > MA20，代表短中期趨勢皆向上。

### 產業輪動分析

用類股指數日報酬與大盤日報酬的差值（RS）計算，每日更新。
分為六個象限，代表各產業目前所處的輪動階段：

| 象限 | 條件 | 意義 |
|------|------|------|
| 準備噴 | 空頭結構，RS 剛翻正，動能轉強 | 底部轉折，注意追蹤 |
| 主升段 | 多頭結構，RS 強，動能加速 | 最強階段，主力資金湧入 |
| 主升回檔 | 多頭結構，短暫回檔，RS 趨勢仍上 | 回檔整理，可找買點 |
| 高檔震盪 | 多頭結構，動能放緩，趨勢轉弱 | 漲多整理，注意出場 |
| 空頭 | 空頭結構，RS 弱，動能持續下滑 | 全面弱勢，迴避 |
| 整理觀察 | 不符任一明確條件 | 過渡中性 |

## 大盤相位過濾器

每日自動判斷加權指數（^TWII）相對 MA20 / MA60 的位置，決定當日允許的訊號類型：

| 大盤狀態 | 條件 | 允許訊號 |
|----------|------|---------|
| 多頭 ▲ | 指數 > MA20 且 > MA60 | 全部訊號 |
| 回檔整理 ↗ | 指數 > MA60，但 < MA20 | 均線回測、縮量回測、假跌破、MA60支撐 |
| 盤整 ─ | 指數介於均線附近 | 均線回測、縮量回測、假跌破、MA60支撐 |
| 空頭 ▼ | 指數 < MA60 × 0.97 | 僅假跌破（強度上限為弱） |

> 大盤相位顯示於「個股選股雷達」頁面頂部狀態列。

## 個股結構標籤

每支股票依均線排列、價格位置、AVWAP 對齊狀況與類股相位，自動標示目前所處結構：

| 結構 | 條件 | 顯示顏色 |
|------|------|---------|
| 主升段✓✓ | MA多頭排列 + 股價≥MA5 + AVWAP三線全對齊 + 類股主升 | 深綠 |
| 主升段✓ | MA多頭排列 + 股價≥MA5 + （類股主升 或 AVWAP三線對齊） | 綠色 |
| 主升段 | MA多頭排列 + 股價≥MA5 | 淺綠 |
| 突破準備✓✓ | 距20日高≤10% + 高於MA20 + RS≥65 + AVWAP三線對齊 + 類股主升 | 深藍 |
| 突破準備✓ | 距20日高≤10% + 高於MA20 + RS≥65 + （類股主升 或 AVWAP三線對齊） | 藍色 |
| 突破準備 | 距20日高≤10% + 高於MA20 + RS≥65 | 淺藍 |
| 回檔 | 多頭排列但股價在MA20～MA10之間，**或**起漲AVWAP跌破 | 橘色 |
| 盤整 | 不符上述條件，股價在MA20之上，或類股空頭降級 | 灰色 |
| 弱勢 | 跌破MA60 或空頭排列 | 紅色 |

> **✓✓** = 類股同步主升 + AVWAP 三線全對齊，為最強結構確認。
> 起漲 AVWAP（`avwap_swing`）跌破時，主升段/突破準備自動降為回檔（趨勢受損）。
> 類股相位為「空頭」時，主升/突破降級為盤整（逆風操作）。

## 買點訊號（fetch_stocks.py）

每支股票最多可同時觸發多個訊號，每個訊號包含：觸發進場價、收盤參考價、MA停損、ATR動態停損、目標價與確認數。

| 訊號名稱 | 觸發條件 | 強度 |
|----------|---------|------|
| 突破 | 收盤 > 20日高，量比 ≥ 1.5x，RS百分位 ≥ 70 | 強 |
| 假跌破 | 昨收跌破20日低，今收強力收復 | 強 |
| 均線回測 | 多頭排列（MA5>MA10>MA20），收盤距MA20在 3% 以內 | 中 |
| 強整再突 | 收盤緊貼20日高（距離 ≤ 5%），收盤 > MA5，RS百分位 ≥ 70 | 中 |
| 回測 | 縮量（量比 < 1.0）回測MA10，收盤 > MA20 | 中 |
| MA60支撐 | 收盤緊貼MA60（距離 ≤ 2%），RS百分位 ≥ 50 | 弱 |

> **最低分數門檻**：綜合分 < 45 的個股不產生任何訊號（過濾弱股假訊號）。
> **觸發進場價**：今日高點 × (1 + buffer)，隔日股價突破此價位才確認進場，避免追高。buffer 依訊號類型不同（突破 0.2%、回測類 0.5%）。
> **停損設定**：突破/強整再突優先以**起漲 AVWAP × 0.99**（主力成本線）為停損；其他訊號以 MA20、MA60、20日低等關鍵支撐為準。另提供 **ATR 14日動態停損**（進場價 − 2×ATR）供參考。
> **目標價計算**：動態風報比（BULL 階段最高 3.0，空頭階段降為 1.0）；AVWAP 三線全對齊時 RR ×1.2，短線 AVWAP 破線時 RR ×0.7。
> **訊號確認數（0～6）**：籌碼分強、RS百分位高、量比大、個股多頭、大盤多頭、**AVWAP 三線全對齊**，符合越多越可信。

## 擴張掃描訊號（fetch_expansion.py）

對台股全市場（約 477 支，過濾條件：股價 ≥ 15、量 ≥ 500張、4碼、非ETF）進行純技術面訊號掃描：

| 訊號名稱 | 觸發條件 | 強度 |
|----------|---------|------|
| 突破 | 收盤 > 20日高，量比 ≥ 1.5x，短線節奏健康 | 強 |
| 假跌破 | 昨收跌破20日低，今收強力收復 | 強 |
| 均線回測 | 多頭排列（MA5>MA10>MA20），收盤距MA20在 3% 以內 | 中 |
| 強整再突 | 收盤緊貼20日高（距離 ≤ 5%），收盤 > MA5，短線節奏健康 | 中 |
| 縮量回測 | 縮量（量比 < 1.0）回測MA10，收盤 > MA20 | 中 |
| MA60支撐 | 收盤緊貼MA60（距離 ≤ 2%） | 弱 |

> 擴張掃描不計算 RS 百分位，故「突破」和「強整再突」無 RS 條件限制。

## Anchored VWAP 三線

兩支掃描程式皆計算三條 AVWAP，用於強化訊號品質判斷：

| 名稱 | 欄位 | 錨點邏輯 | 用途 |
|------|------|---------|------|
| 起漲 AVWAP | `avwap_swing` | 近60日最低點，且後續需有結構確認（price > MA20 或量比 > 1） | 趨勢有沒有壞 |
| 主力 AVWAP | `avwap_vol` | 近20日最大成交量那天，需收盤 > MA20（排除出貨日） | 主力有沒有跑 |
| 短線 AVWAP | `avwap_short` | 近20日最近的局部低點，後3日需有 ≥ 2 根反彈確認 | 短線洗盤節奏 |

**對訊號的影響（兩支程式共同）：**
- `price < 起漲 AVWAP` → 所有訊號強度降一級（strong→medium、medium→weak），reason 標記 ⚠️趨勢破 AVWAP
- `price ≥ 主力 AVWAP` → reason 標記「主力未跑✓」
- `price < 短線 AVWAP` → 跳過突破、強整再突訊號（短線仍在洗盤）

**主掃描額外影響（fetch_stocks.py）：**
- 突破 / 強整再突的**停損優先取 `avwap_swing × 0.99`**（主力成本線），無 AVWAP 時 fallback 至 low20 / MA10
- 三線全對齊時目標價 RR **×1.2**；短線 AVWAP 破時 RR **×0.7**
- 三線全對齊算作第 6 個確認項（確認數最高 6）
- 評分系統含 **AVWAP 對齊分**（0～15，佔綜合分 6%）
- `avwap_swing` 跌破時，個股結構標籤從主升/突破**降為回檔**
- 三線全對齊時，結構標籤升為 **✓✓**（最強確認）
- 三個數值（`avwap_swing`、`avwap_vol`、`avwap_short`）輸出至 `stocks.json`，前端以顏色標示是否站上

## 三法估值（合理價參考）

對每支股票提供保守、合理、樂觀三個估值，取三法中位數：

| 估值方法 | 說明 | 適用條件 |
|---------|------|---------|
| 歷史PE法 | EPS × 該股歷史本益比（低/中/高） | 有歷史PE資料 |
| PEG法 | EPS × EPS成長率（限 3%～30% 區間） | EPS成長率 > 3% |
| 市場PE法 | EPS × 台股平均PE（13x / 17x / 22x） | 所有股票 |

## 回測系統

擴張掃描每次執行時自動對每支股票做 6 個月歷史回測：

- **回測參數**：進場後第 5～15 日判定，風報比固定 2:1
- **backtest_stats**：每次即時計算各訊號類型的歷史勝率（不受 signal_tracking 影響）
- **signal_tracking**：追蹤每筆實際觸發的訊號，最多保留 60 筆，追蹤期最長 20 日

## 類股判斷與對應

系統以三層優先順序判斷個股所屬類股，用於對應產業輪動相位與 RS：

1. **靜態對照表**（`STOCK_SECTOR_MAP`）：硬編碼約 300 支主要股票的正確分類，修正 yfinance 誤標
2. **中文產業名稱對照**（`SECTOR_KEY_MAP`）：yfinance 回傳的中文產業名稱 → 類股 key
3. **英文產業名稱 fallback**（`INDUSTRY_MAP`）：yfinance 回傳英文產業名稱時，先翻譯為中文再查 `SECTOR_KEY_MAP`

> 三層確保大多數股票能取得正確 `sector_key`，進而連結到產業輪動 RS 與相位資料。

## 自動化排程

`.github/workflows/daily_fetch.yml` 設定每個工作日自動執行（台灣時間 15:30）：

- `fetch_stocks.py`：主掃描，抓籌碼 + 基本面 + Gemini 新聞
- `fetch_expansion.py`：擴張掃描，全市場技術訊號 + AVWAP 三線 + 回測

> 15:30 執行是為了確保 TWSE afterTrading API 資料已就緒（市場 13:30 收盤，約需 1.5～2 小時處理）。
> 若 API 仍回傳非 OK 狀態，程式會自動等待重試（最多 4 次，每次間隔 30 秒）。

結果自動推送至 `docs/` 目錄，由 GitHub Pages 對外提供網頁服務。
